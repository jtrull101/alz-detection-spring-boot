Alzheimer's Detection with Machine Learning


This website is built with Angular, interacting with a Spring Boot server application built to support Alzheimer's Disease Detection, leveraging Tensorflow models with Amazon's DJL framework, enabling deep learning support in Java. The purpose of this Machine Learning model is to provide users with an additional form of prediction of impairment level through MRI analysis. Upon choosing a Model from which to run predictions (a default model with 99.5% accuracy is included), the user can submit MRIs to the Angular frontend via the Predictions tab in the navbar, resulting in a readout of predicted level of Impairment due to Alzheimer's Disease.




Disclaimer

The default model included with this site was trained with fictitious MRI data, used to simulate the changes that occur in the brain over the progression of Alzheimer's Disease while not containing any potentially sensitive data. Due to this fact and the nature to which Deep Learning works, the predictions resulting from the default model should NEVER be considered as 100% accurate and true to life. Given a large amount of real MRI data, the model could be improved to eventually remove this disclaimer in the future but that is not possible at this point.




How To Use This Site

Using this site should be straight-forward, but steps are outlined below.

  1. Select a Model - Using the navbar at the top of the screen, click Models. Next, click Choose Active Model. For ease of use, a well performing model is included with this server. For information on how to use your own saved Tensorflow model, scroll down to Using a Saved Tensorflow Model
  2. When the Models have loaded, select one by clicking the associated button. Verify the desired model ID is listed
  3. Submit an MRI - Using the navbar at the top of the screen, click Predictions. Next, click Submit new MRI for Assessment.
  4. Click Browse... to find an MRI image of a brain present on your machine. Note that random predictions can be sent and responses verified using the View Random MRI and View Random MRI by Impairment buttons
  5. With a file selected, click Upload
  6. After a short delay, during which time the MRI image is compressed down to 128x128x3 images (the x3 at the end for RGB channels), then the images are compressed down to raw digit values from 0-255 representing brightness. The values are then normalized to a range from [0,1), and the data is sent to the currently active model for prediction.

Using a Saved Tensorflow Model

  1. Save a Tensorflow Model - Tensorflow models can be saved using the appropriate save function in the language being used. For example, in Python, this method is tensorflow.saved_model.save()
  2. As a result of this saving, all relevant data for the model to again be loaded by Tensorflow is stored in the resulting .zip file
  3. Load Newly Saved Model - Using the navbar at the top of the screen, click Models. Next, click Add New Model.
  4. On the resulting page, click Browse.... Find the model .zip you wish to upload, click Submit, and finally click Upload. Make note of the ID returned.
  5. Select New Model - Using the navbar at the top of the screen, click Models. Next, click Choose Active Model.
  6. Wait for the resulting page to load. Find the model with the specified ID and click its button.
  7. Perform Predictions - Follow the steps described above to perform a prediction on this new model.



Implementation Information

The backend server is implemented with Spring Boot, utilizing an H2 database on disk. The Rest API exposed by this server supports most basic CRUD requests to interact with both loaded Tensorflow models and individual MRI predictions.

The frontend is implemented with Angular, which sends HTTP requests to the Spring Boot server and returns a more beautified response. The user interface is also slightly improved over plain REST requests sent through a RestClient like Postman.
